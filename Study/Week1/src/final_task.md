以下是针对你的 Week1-2 目标的验证测试方案，涵盖功能完整性和结果正确性检查：


### **一、测试准备**
1. **测试数据**：创建一个包含以下特征的测试 CSV 文件（`test_input.csv`），内容示例：
   ```csv
   prot_id,seq,seq_len,pdb_filename,ptm,mean_plddt,emb_filename,label,source
   id1,ABCDEFG（长度=7）,,,...,,positive,
   id2,（长度=1024的随机序列）,,,...,,negative,
   id3,（长度=1025的随机序列）,,,...,,1,
   id4,（长度=500的随机序列）,,,...,,0,
   id5,（长度=2000的随机序列）,,,...,,negative,
   ...（补充足够数据，使总有效样本数为10000，其中训练集8000+测试集2000）
   ```
   - 包含不同长度的序列（<1024、=1024、>1024）
   - 包含多种标签格式（如"positive"/"negative"、1/0等，模拟需要转换的场景）

2. **测试环境**：
   - 确保 Python 环境正常，依赖库（如 `csv`、`argparse` 等）已安装
   - 记录脚本运行时的控制台输出


### **二、功能测试步骤**

#### **1. 基础运行验证**
- **操作**：执行你的 `data_prep.py` 脚本，传入测试输入文件和输出路径，例如：
  ```bash
  python data_prep.py --input test_input.csv --output_train train_out.csv --output_test test_out.csv
  ```
- **预期结果**：
  - 脚本运行无报错（无 `Traceback` 信息）
  - 控制台打印内容包含：`训练集数量：8000，测试集数量：2000`


#### **2. 序列长度过滤验证**
- **操作**：检查输出的 `train_out.csv` 和 `test_out.csv`
- **预期结果**：
  - 所有序列的 `seq_len` 均 ≤ 1024
  - 原输入中长度 >1024 的序列（如 `id3`、`id5`）未出现在输出文件中


#### **3. 标签转换验证**
- **操作**：查看输出文件中的 `label` 列
- **预期结果**：
  - 所有标签均已转换为 `0` 或 `1`（无原标签的其他值，如"positive"、"negative"等）
  - 转换逻辑合理（例如：原"positive"→1，"negative"→0；原1→1，0→0）


#### **4. 数量匹配验证**
- **操作**：
  1. 统计 `train_out.csv` 中数据行数量（排除表头）
  2. 统计 `test_out.csv` 中数据行数量（排除表头）
- **预期结果**：
  - 训练集行数 = 8000
  - 测试集行数 = 2000
  - 总有效行数（训练+测试）= 输入中所有 `seq_len ≤ 1024` 的样本数（需提前计算输入中符合条件的样本总数）


### **三、边界条件测试**
1. **极端长度序列**：
   - 输入包含 `seq_len=1024` 的序列 → 应被保留
   - 输入包含 `seq_len=1025` 的序列 → 应被过滤

2. **空标签或异常标签**：
   - 若输入存在空标签（`label=""`）或未定义标签（如"unknown"），脚本应能处理（例如默认转为0或1，或报错提示需预处理，需符合你的设计逻辑）

3. **输入数据量不足**：
   - 若输入中符合条件的样本总数 <10000（例如仅5000个），脚本应打印实际数量（而非强制8000/2000），或按比例拆分（需符合你的设计逻辑）


### **四、通过标准**
1. 脚本运行无报错。
2. 控制台准确打印 `训练集数量：8000，测试集数量：2000`（在输入数据足够的前提下）。
3. 输出文件中无长度 >1024 的序列。
4. 输出文件的标签列仅包含 `0` 和 `1`。
5. 训练集和测试集的实际行数与打印数量一致。

若全部满足，则说明你的目标已达标！
